# Модули — готовые флоу и паки узлов

Модуль — это каталог в [open-lzt/lzt-flows](https://github.com/open-lzt/lzt-flows). Ставится из бота:

```
/modules            — что есть
/import bump-daily  — поставить
```

Есть две разновидности, и разница между ними — не формат, а доверие.

| | `kind: flow` | `kind: python` |
|---|---|---|
| Что это | граф из узлов, которые у движка уже есть | пак новых узлов |
| Содержит | `module.yaml`, `flow.json` | `module.yaml`, `pyproject.toml`, `*.py` |
| Кто публикует | **кто угодно** | **только владелец репозитория** |
| Как ставится | `/import` в боте — API проверит и сохранит | руками на машине: `pip install` + рестарт воркера |
| Почему так | худшее, что может граф, — то, что могут его узлы, а их валидатор проверяет все | это чужой код в вашем воркере, с вашими токенами и деньгами |

**API не ставит код никогда.** `/import` для `kind: python` отклоняется — до скачивания байтов.
Иначе установка пака была бы удалённым выполнением кода, доступным любому обладателю API-ключа, а
это ключ бота, а это один угнанный телеграм-аккаунт.

## Опубликовать флоу

Один раз добавьте себя в `authors.yml` — **отдельным PR**. CI отклонит PR, который трогает и
`authors.yml`, и модуль: заявку на личность и флоу, за который она ручается, не проверяют одним
взглядом.

Затем PR с `modules/<имя>/`:

**`module.yaml`**
```yaml
schema_version: 1
name: bump-daily
version: 1.0.0
author: ваш-github-логин    # должен совпасть с автором PR
description: Поднимает все лоты аккаунта.
requires_nodes:
  - logic.get_my_lots
  - logic.for_each_lot
  - market.bump
```

**`flow.json`** — граф. Проверьте локально тем же валидатором, что и CI:

```bash
uv run lzt-flow-validate modules/bump-daily
```

Прошло — установится. Не прошло — увидите причину в PR, а не потом. Это одна и та же функция; двух
реализаций нет, потому что они разойдутся, и в день расхождения CI пропустит то, что бэкенд всё
равно запустит.

`index.json` не трогайте — он пересобирается на main. PR с ним отклоняется.

## За что модуль отклонят

| Причина | Что случилось |
|---|---|
| `unknown_node` | ссылается на узел, которого у движка нет |
| `forbidden_capability` | использует узел с `reflective` — произвольный вызов API маркета |
| `compile_failed` | граф не проходит **настоящий** компилятор |
| `code_in_module` | `.py` в `kind: flow`, или посторонний файл в паке |
| `bad_name` | имя не `[a-z0-9-]` — оно становится сегментом пути |
| `checksum_mismatch` | байты не те, что были отревьюены |

Фильтр по **возможностям**, а не по именам: рефлексивный узел, добавленный через месяц, поймается
сам, без правки списка.

## Честно про доверие

**`sha256` — это целостность передачи.** Он доказывает, что флоу у вас байт-в-байт тот, который
отревьюили. Это **не подпись** и он ничего не говорит о том, можно ли доверять автору.

Единственное, что защищает вас от вредного модуля, — то, что мейнтейнер прочитал его перед мержем.
`flow.json` — обычный JSON: читайте сами, там видно всё, что модуль сделает.

**CI не является границей.** Для PR из форка GitHub берёт workflow из мержа head в base — автор PR
может отредактировать `validate.yml` в своём же PR и снять проверку. Секретов и write-токена он не
получит, но проверку снимет. Код в main держат **CODEOWNERS + branch protection**: файл попадает
туда потому, что владелец нажал merge. CI-проверка нужна, чтобы ревьюер понял, почему PR
подозрительный, до чтения четырёхсот строк.

Бэкенд тоже не верит CI: при импорте модуль **перепроверяется заново** против реестра узлов
*этого* процесса. CI проверял против того набора, что был у раннера в момент мержа; у вас может
быть другой — узел сняли, плагин удалили. Реестр знает, что было безопасно *там и тогда*; только
ваш процесс знает, что запускаемо *здесь и сейчас*. Иначе это TOCTOU с платным действием в конце.

## Если реестр недоступен

Список будет **пустым**, а не устаревшим. Пустой каталог — заметно деградировавший интерфейс.
Устаревший-непроверенный каталог выглядит рабочим и предлагает модули, целостность которых никто не
сверял. Незаметный отказ — опаснее.

Бот скажет «модулей нет **или** реестр недоступен» — он не различает эти случаи намеренно.

## Опубликовать пак узлов

Только владелец репозитория. Каталог с `module.yaml` (`kind: python`), `pyproject.toml` и пакетом.
Примеры: `modules/notify-pack/`, `modules/pricing-pack/`.

Как писать сами узлы — [Плагины](plugins.md). Ставится на машине:

```bash
pip install "lzt-flow-notify-pack @ git+https://github.com/open-lzt/lzt-flows.git#subdirectory=modules/notify-pack"
sudo systemctl restart open-lzt-flow-worker open-lzt-flow-api
```

Узлам с сетью не забудьте открыть хост:

```bash
LZT_FLOW_EGRESS_ALLOWED_HOSTS=api.telegram.org,discord.com
```
